# dietarysupplement.ai supplement scorer — monorepo

<a name="english"></a>

A **deterministic supplement scoring capability**, delivered through three calling layers: Claude Skill, ChatGPT, and a browser-based agent.
All categories, brand-agnostic, global. Scoring runs a deterministic engine (not LLM-estimated); models only handle conversation and explanation.

## Single source of truth (the key idea)

**Only edit `engine/`, then run `python build.py` to sync to all layers.** Do not hand-edit any "generated" file.

```
engine/
├── data.json          ← single data source: reference tables + weights (edit here)
├── score.py           ← single algorithm (edit here)
└── test_engine.py     ← assertion tests
build.py               ← sync tool: pushes engine → the three layers below
claude-skill/
└── supplement-scorer/ ← Claude Skill (SKILL.md + generated scripts/score.py + references/)
chatgpt/
├── score.py           ← generated: self-contained (data inlined, for Code Interpreter)
├── gpt-instructions.md, openapi.yaml, server.py, dev-doc.md
web/
└── index.html         ← browser-based agent (CFG + REFERENCE data block generated by build.py)
```

After editing:

```bash
python build.py                  # sync all four locations
python engine/test_engine.py     # assertion tests (7/7)
```

> **Verification**: the same "premium iron supplement" scores **97** across engine tests, `chatgpt/score.py`, `claude-skill`, and `web`. Change weights or dose ranges in `engine/data.json`, build, and all four update together.

## How to use each layer

- **Claude Skill** — package `claude-skill/supplement-scorer/` and upload to Claude.ai (Settings → Capabilities → Skills), or drop it into the Claude Code skills directory.
- **ChatGPT** — follow `chatgpt/dev-doc.md`: Custom GPT uploads `chatgpt/score.py` + enable Code Interpreter + paste `gpt-instructions.md`; for stability, use `server.py` + `openapi.yaml` as an Action; for your own product, use the Responses API.
- **Web** — `web/index.html` is an interactive browser-based agent (runs directly in the Claude.ai artifact environment; standalone deployment requires a backend proxy to hold the model API key).

## Caveats

- The reference values in `engine/data.json` are a **v0 international baseline, manually entered, non-authoritative**. National RDAs/ULs and permitted claims vary; before going live, a domain expert must review and tag each value with a source per market.
- The safety-deduction constants (25 / 15) are hardcoded in the engine algorithm (algorithm constants, not data).
- Web is JS, the rest is Python: `build.py` syncs **data** (from `data.json`); the algorithm itself is maintained on both sides as a mirror. When changing algorithm logic, update both.

## 各层怎么用

- **Claude Skill** —— 打包 `claude-skill/supplement-scorer/` 上传到 Claude.ai(Settings → Capabilities → Skills),或放进 Claude Code 的 skills 目录。
- **ChatGPT** —— 按 `chatgpt/dev-doc.md`:Custom GPT 传 `chatgpt/score.py` + 开 Code Interpreter + 粘 `gpt-instructions.md`;要稳就用 `server.py` + `openapi.yaml` 做 Action;接自家产品用 Responses API。

## 注意

- `engine/data.json` 的参考值是 **v0 国际基线、手工录入、非权威**;各国 RDA/UL 与可声称项不同,上线前需专业人士复核并按市场挂来源。
- 安全扣分常数(25/15)在引擎算法里写死(算法常数,非数据)。
- Web 是 JS、其它是 Python:`build.py` 同步的是**数据**(从 `data.json`);两边的算法各自维护、保持镜像。改算法逻辑时记得两处都改。
